## 1. 背景・目的

フェーズ1 により、Slack 上の重要メッセージが Notion に自動蓄積されるようになった。
フェーズ2 では、この蓄積データを AI で検索・回答できる仕組みを構築する。

**解決する課題**: 「以前のミーティングで決まったあの話、何だったっけ？」という情報検索が Slack だと困難。Notion に蓄積された社内ナレッジを自然言語で即座に検索・回答できる状態にする。

**実現する体験**:
```
@knowledgeBot 新規顧客への提案で使えるアイデアは？

→ ボットが Notion の保存済みナレッジから関連情報を取得し、
  「過去のアイデアとして〇〇が挙げられています（出典: #general 2026-03-05）」
  と返答する
```

> すでに使い慣れているナレッジ保存ボットへのメンション形式で自然に質問できる。
> スラッシュコマンドを覚える必要がなく、チャンネル上でのやり取りとして会話に残る。

## 2. スコープ

### 2.1 フェーズ2 の対象

| 機能 | 内容 |
|------|------|
| ベクトル埋め込み自動更新 | Notion にナレッジが保存されるたび、そのテキストをベクトル化して Upstash Vector に保存 |
| @メンションによる質問受付 | `@knowledgeBot <質問>` をトリガーに、Gemini AI が回答を生成して返す |
| 類似ナレッジ検索 | 質問と意味的に近いナレッジを Upstash Vector から取得する（RAG の Retrieval 部分） |
| 回答生成 | 取得したナレッジを元に Gemini Flash が回答を生成する（RAG の Generation 部分） |

### 2.2 フェーズ2 で対応しないこと

- Notion の既存ページを一括でベクトル化するバッチ処理（初回は手動実行で対応）
- Notion DB に保存されていないデータへの回答
- リアルタイム Web 検索との連携
- 回答履歴の保存・会話コンテキストの維持

## 3. システム概要

### 3.1 RAG アーキテクチャ

```
【蓄積フロー（フェーズ1 からの拡張）】

Slack リアクション
  ↓
Notion DB に保存（フェーズ1 既存）
  ↓
Gemini Embedding API（gemini-embedding-001）
  ↓ テキストをベクトル化
Upstash Vector
  ↓ ベクトルを保存


【検索・回答フロー（フェーズ2 新規）】

@knowledgeBot <質問>
  ↓ app_mention イベント → POST /api/slack/events（既存エンドポイントで処理）
Gemini Embedding API
  ↓ 質問をベクトル化
Upstash Vector（ANN 類似検索）
  ↓ 上位 5 件を取得
Gemini Flash（gemini-2.5-flash）
  ↓ プロンプト + コンテキストで回答生成
Slack chat.postMessage
  ↓ 回答をメンションのスレッドへ返信
```

**ポイント**: 検索・回答フローも既存の `/api/slack/events` エンドポイントで処理する。新しいエンドポイントは不要。

### 3.2 処理フロー詳細

**蓄積フロー（`events/route.ts` の `reaction_added` 処理に追加）**

| ステップ | 内容 |
|----------|------|
| ① | フェーズ1 の処理で Notion ページ作成が完了する |
| ② | Notion ページの本文テキストを Gemini Embedding API でベクトル化 |
| ③ | ベクトルとメタデータ（title, channel, savedAt, notionUrl, category）を Upstash Vector に upsert |

**検索・回答フロー（`events/route.ts` に `app_mention` ハンドラを追加）**

| ステップ | 内容 |
|----------|------|
| ① | ユーザーが `@knowledgeBot <質問>` をチャンネルに投稿 |
| ② | Slack が `app_mention` イベントを POST `/api/slack/events` に送信 |
| ③ | Slack 署名検証（既存の `verifySlackSignature` を流用） |
| ④ | イベントタイプが `app_mention` の場合、メンション部分を除いた質問テキストを抽出 |
| ⑤ | 質問テキストを Gemini Embedding API でベクトル化 |
| ⑥ | Upstash Vector に類似検索リクエストを送信（TopK=5） |
| ⑦ | 取得したナレッジをコンテキストとして Gemini Flash にプロンプトを送信 |
| ⑧ | 生成された回答を、メンションされたメッセージのスレッドへ返信 |
| ⑨ | 200 OK を返す |

### 3.3 技術スタック

| レイヤー | 採用技術 | 備考 |
|----------|----------|------|
| フレームワーク | Next.js 14 (App Router) | フェーズ1 と同一 |
| ホスティング | Vercel (Hobby プラン) | フェーズ1 と同一 |
| 埋め込みモデル | Gemini Embedding API（`gemini-embedding-001`） | 1536次元ベクトルを生成 |
| LLM | Gemini Flash（`gemini-2.5-flash`） | 回答生成。低コスト・高速 |
| ベクトル DB | Upstash Vector | 既存 Upstash アカウントで追加作成 |
| Slack SDK | `@slack/web-api` | フェーズ1 と同一 |

**Gemini を選定した理由**

1. `gemini-2.5-flash` はコストが低く、社内ナレッジ検索程度の用途に最適
2. Notion → Gemini のデータパイプラインが公式ドキュメントで整備されている
3. 将来的に Google Workspace（Gmail / Google Drive）連携に発展させやすい

## 4. 機能要件

### 4.1 ベクトル埋め込み自動更新機能

| No. | 機能名 | 詳細 |
|-----|--------|------|
| F2-01 | ページ保存時の自動ベクトル化 | Notion にナレッジが保存されるたびに（フェーズ1 の処理フローの末尾で）、`fullText` を Gemini Embedding API に送信してベクトルを取得する |
| F2-02 | Upstash Vector への upsert | 取得したベクトルと以下のメタデータを Upstash Vector に upsert する: `notionPageId`（主キー）、`title`、`channel`、`savedAt`、`notionUrl`、`category` |
| F2-03 | 埋め込みエラーの分離 | Gemini Embedding API または Upstash Vector への書き込みに失敗しても、Notion 保存・Slack 通知（フェーズ1 の動作）には影響を与えない |

### 4.2 @メンション質問応答機能

| No. | 機能名 | 詳細 |
|-----|--------|------|
| F2-04 | メンション検知 | `app_mention` イベントを検知する。ボット自身の投稿によるメンションは無視する（無限ループ防止） |
| F2-05 | 質問テキスト抽出 | メッセージテキストからメンション部分（`<@BOTID>`）を除いた文字列を質問テキストとして抽出する |
| F2-06 | 空質問対応 | 質問テキストが空の場合は「使い方: `@knowledgeBot <質問内容>` と話しかけてください」と返す |
| F2-07 | 署名検証 | フェーズ1 と同じ `verifySlackSignature` 関数で検証する（既存処理と共通） |
| F2-08 | 質問のベクトル化 | 質問テキストを Gemini Embedding API でベクトル化する |
| F2-09 | 類似ナレッジ検索 | Upstash Vector に対し TopK=5 で類似検索を行う |
| F2-10 | 検索結果なし対応 | 類似ナレッジが見つからない場合（スコアが閾値未満 or 件数0）は「関連する情報が見つかりませんでした」と返す |
| F2-11 | 回答生成 | 取得したナレッジをコンテキストとして Gemini Flash に送信し、回答を生成する |
| F2-12 | スレッドへの返信 | 生成された回答と参照したナレッジの出典（タイトル・チャンネル・Notion URL）を、メンションされたメッセージのスレッドに返信する |
| F2-13 | 3秒以内の ACK | Slack は3秒以内の応答を要求する。処理は同期で完結させる（Vercel Hobby タイムアウト10秒以内） |

### 4.3 回答フォーマット仕様

**Slack 返信例（Block Kit・スレッドへの返信）**

```
新規顧客提案で活用できるアイデアとして、過去に以下の内容が挙げられています：
・○○の機能を前面に出したデモが効果的との意見がありました。
・△△業界向けの事例紹介を準備すると刺さりやすいとのこと。

参照したナレッジ:
📌 新規顧客向けデモ構成の検討 (#sales / 2026-02-15) → [Notion で開く]
💡 提案資料に入れるべき事例リスト (#general / 2026-01-20) → [Notion で開く]
```

## 5. Upstash Vector 設計

### 5.1 インデックス設定

| 項目 | 設定値 |
|------|--------|
| Dimensions | 1536（`gemini-embedding-001` の出力次元 / `outputDimensionality` で削減） |
| Similarity Metric | Cosine |

### 5.2 ベクトルレコード構造

```json
{
  "id": "notion_page_id（Notion ページ ID をそのまま使用）",
  "vector": [0.012, -0.034, ...],
  "metadata": {
    "title": "新規顧客向けデモ構成の検討",
    "category": "アイデア",
    "channel": "#sales",
    "savedAt": "2026-02-15T10:30:00.000Z",
    "notionUrl": "https://notion.so/YOUR_PAGE_ID"
  }
}
```

### 5.3 類似検索パラメータ

| パラメータ | 値 |
|-----------|-----|
| TopK | 5 |
| スコア閾値 | 0.75（コサイン類似度。これ未満は「関連情報なし」と判断） |
| includeMetadata | true |

## 6. Gemini プロンプト設計

### 6.1 システムプロンプト

```
あなたは社内ナレッジ検索アシスタントです。
以下のナレッジ一覧はSlackで重要とマークされ保存された社内情報です。
ユーザーの質問に対して、ナレッジの内容を参照して日本語で簡潔に回答してください。

【ルール】
- ナレッジに記載のない情報は回答に含めないこと
- 回答は3〜5文程度にまとめること
- 推測・憶測は行わないこと
```

### 6.2 ユーザープロンプト（動的生成）

```
【ナレッジ一覧】
---
タイトル: {title}
カテゴリ: {category}
チャンネル: {channel}
保存日: {savedAt}
内容:
{fullText（先頭 1000 文字）}
---
（以上を最大5件繰り返す）

【質問】
{ユーザーの質問テキスト}
```

**⚠️ 実装上の注意点**

- `app_mention` イベントにはボット自身へのメンションも含まれる。`event.bot_id` が存在する場合は処理をスキップし、無限ループを防ぐこと。
- Gemini API と Upstash Vector の呼び出しは逐次実行になるため、合計3秒を超える可能性がある。Vercel Hobby の10秒タイムアウト内に収まるか本番で要確認。
- Vercel の環境変数追加後は必ず Redeploy すること。

## 11. エラーハンドリング方針

| エラー発生箇所 | 対応 |
|---------------|------|
| Gemini Embedding 失敗（蓄積フロー） | エラーログを出力し、フェーズ1 の処理（Notion 保存・Slack 通知）には影響させない |
| Upstash Vector upsert 失敗 | エラーログを出力し、フェーズ1 の処理には影響させない |
| Gemini Embedding 失敗（検索フロー） | 「検索に失敗しました。しばらく待ってから再試行してください。」とスレッドへ返信する |
| Upstash Vector 検索失敗 | 同上 |
| Gemini Flash 回答生成失敗 | 「回答の生成に失敗しました。」とスレッドへ返信する |
| 関連ナレッジ 0 件（スコア閾値未満） | 「関連する情報が見つかりませんでした。」とスレッドへ返信する |
| 署名検証失敗 | 403 を返す（フェーズ1 と同一） |
| ボット自身のメンション | 無視する（処理せず 200 を返す） |
