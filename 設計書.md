## 2. エンドポイント設計

### 2.1 `POST /api/slack/events`

| 項目 | 内容 |
|------|------|
| パス | `/api/slack/events` |
| メソッド | POST |
| 呼び出し元 | Slack Events API |
| 認証 | `X-Slack-Signature` による HMAC-SHA256 署名検証 |

**リクエストボディ（Slack `reaction_added` イベント）**

```json
{
  "type": "event_callback",
  "event": {
    "type": "reaction_added",
    "reaction": "pushpin",
    "user": "U012AB3CD",
    "item": {
      "type": "message",
      "channel": "C024BE91L",
      "ts": "1360782804.083113"
    },
    "item_user": "U089AB4CD",
    "event_ts": "1360782804.083113"
  }
}
```

**レスポンス**

| 条件 | ステータス | ボディ |
|------|-----------|--------|
| URL Verification challenge | 200 | `{ challenge: "..." }` |
| 処理成功 | 200 | `{ ok: true }` |
| 署名検証失敗 | 403 | `{ error: "invalid_signature" }` |
| 重複イベント | 200 | `{ ok: true, skipped: true }` |
| 対象外リアクション | 200 | `{ ok: true, skipped: true }` |
| サーバーエラー | 500 | `{ error: "internal_error" }` |

## 3. モジュール設計

### 3.1 `app/api/slack/events/route.ts`

メインの処理オーケストレーターとして機能する。

```typescript
// ファイル先頭に必須（ないとビルドエラー）
export const dynamic = 'force-dynamic'

// 処理フロー（擬似コード）
export async function POST(req: Request) {
  // 1. Slack URL Verification への対応
  // 2. 署名検証（lib/slack.ts）
  // 3. reaction_added 以外は無視
  // 4. 対象絵文字（pushpin/bulb/white_check_mark）以外は無視
  // 5. Redis 重複チェック（lib/redis.ts）
  // 6. Slack スレッド全文取得・チャンネル名・投稿者名を並列取得（lib/slack.ts）
  // 7. Notion ページ作成 と Redis 書き込みを並列実行
  // 8. Slack スレッド返信（lib/slack.ts）
  // 9. 200 OK 返却
}
```

**重要①**: `export const dynamic = 'force-dynamic'` は必須。Next.js 14 はビルド時にルートを静的解析するが、このディレクティブがないと環境変数未解決でビルドエラーになる。

**重要②**: Slack は3秒以内に 200 を返さないとリトライする。処理を軽量に保ち同期処理で完結させる。

## 5. エラーハンドリング方針

| エラー発生箇所 | 対応 |
|---------------|------|
| 署名検証失敗 | 即時 403 を返す。以降の処理は実行しない。 |
| 重複チェック（Redis 接続失敗） | エラーログを出力し、重複と見なして 200 を返す（安全側に倒す）。 |
| Slack API 失敗（スレッド取得） | エラーログを出力し、500 を返す。 |
| Notion API 失敗 | エラーログを出力し、500 を返す。Redis への書き込みは行わない（リトライ可能にする）。 |
| Slack 返信失敗（通知送信） | エラーログを出力するが、Notion 保存は成功しているため 200 を返す。 |

## 6. Slack App 設定

### 6.1 必要な OAuth スコープ（Bot Token Scopes）

| スコープ | 用途 |
|---------|------|
| `reactions:read` | `reaction_added` イベントの受信 |
| `channels:history` | パブリックチャンネルのスレッド取得 |
| `groups:history` | プライベートチャンネルのスレッド取得 |
| `im:history` | 1対1 DM のスレッド取得 |
| `mpim:history` | グループ DM のスレッド取得 |
| `chat:write` | 保存完了通知の返信送信 |
| `users:read` | 投稿者の表示名取得 |

### 6.2 Event Subscriptions 設定

| 項目 | 値 |
|------|----|
| Request URL | `https://<your-vercel-domain>/api/slack/events`（`/api/slack/events` まで含める） |
| Subscribe to bot events | `reaction_added` |

### 6.3 ボットのチャンネル参加

**ボットが参加していないチャンネルのリアクションはイベントが届かない。** 使用するチャンネルで以下を実行：

```
/invite @ボット名
```

```bash
# 1. 依存インストール（package.json に全パッケージ記載済み）

ngrok http 3000
# → 表示された https://xxxx.ngrok-free.app をコピー

# 5. Slack App の Request URL を更新
# https://api.slack.com/apps → Event Subscriptions

```

**注意**: ngrok を再起動するたびに URL が変わるため、Slack App の Request URL を更新し直す必要がある。

## 8. Vercel デプロイ設定

| 項目 | 設定値 |
|------|--------|
| Framework Preset | Next.js |
| Node.js Version | 20.x |
| 環境変数 | Vercel ダッシュボードの Environment Variables に `.env.local` の内容を設定 |
| Region | hnd1（東京）推奨 |

**デプロイ前チェックリスト（実装時に判明した注意点）**

| チェック項目 | 詳細 |
|-------------|------|
| `export const dynamic = 'force-dynamic'` | `route.ts` の先頭に必須。ないとビルドエラー |
| Vercel 環境変数 | 6変数を全て設定し、設定後に Redeploy する |
| Slack Request URL | `https://xxxx.vercel.app/api/slack/events`（`/api/slack/events` まで含める） |
| ボットのチャンネル招待 | 対象チャンネルで `/invite @ボット名` を実行。招待しないとイベントが届かない |

**注意**: Vercel Hobby プランの Serverless Function タイムアウトは **10秒**。Slack の3秒応答要件を守るため、処理のボトルネックを避ける。特に `conversations.replies` と Notion API の呼び出しは逐次実行ではなく `Promise.all` で並列化する。

```typescript
// ❌ 逐次実行（遅い）
const messages = await fetchThreadMessages(channelId, ts)
const page = await createKnowledgePage(data)

// ✅ 並列化（ただし messages が page 作成に必要なため不可）
// → messages 取得後に Notion 保存。Notion 保存と Redis 書き込みを並列化する。
const [_, notionResult] = await Promise.all([
  markAsProcessed(redisKey),
  createKnowledgePage(data),
])
```

## 9. テスト方針

| レイヤー | 手法 |
|---------|------|
| 署名検証 | 無効な `X-Slack-Signature` で 403 が返ることを curl で確認 |
| 重複防止 | 同一ペイロードを2回 POST して Notion が1件のみ作成されることを確認 |
| カテゴリ分岐 | 3種の絵文字それぞれで Notion の Category が正しく設定されることを確認 |
| 通知返信 | Slack チャンネルにボット返信が届き、Notion URL が含まれることを確認 |
| エラー系 | Notion API を意図的に失敗させてボット返信が送信されないことを確認 |
