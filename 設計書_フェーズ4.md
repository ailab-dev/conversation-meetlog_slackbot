# 設計書 — フェーズ4 チャンネル参加時データ収集

最終更新: 2026-03-10

---

## 1. 全体フロー

```
① /invite @knowledgeBot
      │
      ▼
② member_joined_channel イベント
   → /api/slack/events
   → ボット自身の参加か確認
   → 確認メッセージ投稿（Block Kit ボタン）

③ ユーザーが [はい] をクリック
      │
      ▼
④ block_actions ペイロード
   → /api/slack/actions
   → 即座に 200 返却（Slack タイムアウト回避）
   → ボタンを「収集開始します...」に更新
   → Redis に進行中フラグをセット
   → Upstash QStash にジョブを発行

⑤ QStash → /api/slack/collect（バックグラウンド・繰り返し）
   → conversations.history で 100件ずつ取得
   → 全メッセージを処理（5文字未満・bot メッセージはスキップ）
   → Notion + Vector 保存
   → 続きがあれば cursor 付きで次ジョブを自己発行
   → 完了時に Redis フラグ削除 + Slack へ完了通知
```

---

## 2. 保存対象の切り替え

```
一括収集（フェーズ4）     : 全メッセージ対象（リアクション不問）
リアルタイム（フェーズ1） : リアクション付きのみ（変更なし）
```

リアルタイムの保存条件（`/api/slack/events`）は既存のまま変更しない。

## 3. 新規エンドポイント

### 3-1. /api/slack/actions（新規）

Block Kit ボタンのインタラクションを受け取る。

```typescript
// app/api/slack/actions/route.ts
export async function POST(req: NextRequest): Promise<NextResponse> {
  // Slack Interactive Components は application/x-www-form-urlencoded で届く
  const formData = await req.formData()
  const payload = JSON.parse(formData.get('payload') as string)

  // 署名検証（verifySlackSignature を流用）
  const actionId = payload.actions[0].action_id
  const channelId = payload.channel.id

  if (actionId === 'collect_yes') {
    // 多重起動チェック
    const isRunning = await isCollecting(channelId)
    if (isRunning) {
      await updateMessage(payload, '現在収集中です。完了までお待ちください。')
      return NextResponse.json({ ok: true })
    }
    // 進行中フラグをセット（最大 2時間）
    await setCollecting(channelId, 7200)
    // ボタンを更新
    await updateMessage(payload, '📥 収集を開始しました。完了後に通知します。')
    // QStash にジョブを発行
    await publishCollectJob({ channelId, cursor: null, collected: 0 })

  } else if (actionId === 'collect_no') {
    await updateMessage(payload, 'キャンセルしました。')
  }

  return NextResponse.json({ ok: true })
}
```

### 3-2. /api/slack/collect（新規）

QStash から呼び出されるバックグラウンド処理エンドポイント。

```typescript
// app/api/slack/collect/route.ts
interface CollectJob {
  channelId: string
  cursor:    string | null
  collected: number  // 累計保存件数
}

export async function POST(req: NextRequest): Promise<NextResponse> {
  // QStash 署名検証
  const receiver = new Receiver({
    currentSigningKey: process.env.QSTASH_CURRENT_SIGNING_KEY!,
    nextSigningKey:    process.env.QSTASH_NEXT_SIGNING_KEY!,
  })
  await receiver.verify({ signature: req.headers.get('upstash-signature') ?? '', body: rawBody })

  const job: CollectJob = JSON.parse(rawBody)

  // conversations.history で 1ページ（100件）取得
  const result = await slack.conversations.history({
    channel: job.channelId,
    cursor:  job.cursor ?? undefined,
    limit:   100,
  })

  let collected = job.collected
  for (const msg of result.messages ?? []) {
    // bot メッセージ・短すぎるメッセージはスキップ
    if (msg.bot_id) continue
    const text = (msg.text ?? '').trim()
    if (text.length < 5) continue

    // 重複チェック → Notion + Vector 保存（リアクション不問）
    collected += await processSingleMessage(job.channelId, msg)
  }

  if (result.has_more && result.response_metadata?.next_cursor) {
    // 続きを QStash に積む（チェーン）
    await publishCollectJob({
      channelId: job.channelId,
      cursor:    result.response_metadata.next_cursor,
      collected,
    })
  } else {
    // 収集完了
    await clearCollecting(job.channelId)
    await postCollectComplete(job.channelId, collected)
  }

  return NextResponse.json({ ok: true })
}
```

## 6. processSingleMessage の設計

`batch-import.ts` のメッセージ処理ロジックを `lib/collect.ts` に切り出して共通化する。

```typescript
// lib/collect.ts
export async function processSingleMessage(
  channelId: string,
  msg: Record<string, unknown>
): Promise<number> {
  const ts       = msg.ts as string
  const rawText  = msg.text as string ?? ''
  const userId   = msg.user as string ?? ''

  // Redis 重複チェック（キー: knowledge:{channelId}:{ts}:_bulk）
  const redisKey = buildRedisKey(channelId, ts, '_bulk')
  if (await isDuplicate(redisKey)) return 0

  // スレッド全文取得
  const replies = await fetchThreadMessages(channelId, ts)
  let fullText  = replies.map(m => m.text).filter(Boolean).join('\n\n---\n\n')

  // フェーズ3エンリッチメント
  const files = await fetchMessageFiles(channelId, ts)
  const enrichment = await buildEnrichment(rawText, files, process.env.SLACK_BOT_TOKEN!)
  if (enrichment) fullText += enrichment

  // Notion 保存
  const postedBy  = await getUserDisplayName(userId)
  const channelName = await getChannelName(channelId)
  const savedAt   = new Date(parseFloat(ts) * 1000).toISOString()
  const title     = cleanText(rawText).slice(0, 80) || '（本文なし）'

  const notionResult = await createKnowledgePage({
    title, category: '一般', postedBy, slackChannel: channelName, savedAt, fullText,
  })
  await markAsProcessed(redisKey)

  // Vector 保存
  const vector = await embedText(fullText)
  await upsertVector(notionResult.id, vector, {
    title, category: '一般', channel: channelName, savedAt,
    notionUrl: notionResult.url, fullText: fullText.slice(0, 1000),
  })

  return 1
}
```

**Redis キー設計の注意点**

一括収集専用のサフィックス `_bulk` を使用することで、
リアクション時の Redis キー（`knowledge:{ch}:{ts}:pushpin` 等）と競合しない。

## 7. Upstash QStash によるキュー設計

### 採用理由

Vercel Serverless タイムアウト（最大 10〜60秒）内にチャンネル全履歴を処理できないため。
100件 × 各処理（Slack/Notion/Gemini API）で数分かかる可能性がある。

### ジョブチェーン

```
/api/slack/actions
  └─ QStash.publish → /api/slack/collect { cursor: null, collected: 0 }
                          └─ 100件処理
                          └─ QStash.publish → /api/slack/collect { cursor: "xxx", collected: 87 }
                                                 └─ 100件処理
                                                 └─ ... (繰り返し)
                                                 └─ 完了 → Slack通知
```

### 多重起動防止（Redis フラグ）

```
setCollecting(channelId)    → Redis.set('collecting:{channelId}', '1', { ex: 7200 })
isCollecting(channelId)     → Redis.exists('collecting:{channelId}')
clearCollecting(channelId)  → Redis.del('collecting:{channelId}')
```
