# 設計書 フェーズ6：タスク管理機能

## アーキテクチャ概要

```
Slack メッセージ
  ↓ (自動収集済み)
events/route.ts → detectTaskKeywords()
  → ヒット → notifyTaskCandidate() → 確認ボタン送信 → Redis に task_pending 保存

「タスクに追加」ボタン押下
  ↓
actions/route.ts
  → Redis から task_pending 取得
  → Gemini で期日抽出
  ─ 期日あり → Notion 更新 → task_meta / task_channel / task_user を Redis 保存 → 完了通知
  └ 期日なし → メッセージを日付ピッカー UI に更新 → Redis に task_date_pending 保存

「この日付で登録」or「日付なしで登録」ボタン押下
  ↓
actions/route.ts
  → Redis から task_date_pending 取得 → 選択日付と合わせて Notion 更新 → 完了通知

毎朝 9:00 JST (Vercel Cron UTC 0:00)
  ↓
/api/cron/remind
  → Notion から DueDate 0〜2日以内の タスク を取得
  → Redis で reminder_sent チェック
  → Slack に @メンション付きリマインド送信
  → Redis に reminder_sent をマーク

スレッド返信（完了キーワード含む）
  ↓
events/route.ts → detectCompletionKeywords()
  → Redis で task_meta:{channelId}:{parentTs} を確認
  → タスク登録済みなら → Notion Category を 完了 に更新 → 完了通知
```

---

## Redis キー設計

| キー | 値 | TTL | 用途 |
|-----|-----|-----|------|
| `task_pending:{channelId}:{confirmMsgTs}` | JSON `{notionPageId, rawText, userId}` | 3日 | タスク追加確認ボタン待ち |
| `task_date_pending:{channelId}:{dateMsgTs}` | JSON `{notionPageId, userId}` | 3日 | 日付ピッカーボタン待ち |
| `task_meta:{channelId}:{originalMsgTs}` | JSON `{notionPageId, notionUrl, title}` | 90日 | タスク登録済み判定・完了検知 |
| `task_channel:{notionPageId}` | `channelId` | 90日 | リマインダー送信先チャンネル |
| `task_user:{notionPageId}` | `slackUserId` | 90日 | @メンション用ユーザー ID |
| `reminder_sent:{notionPageId}:{type}` | `1` | 5日 | 重複送信防止 (type: `2d`/`1d`/`0d`) |

## Block Kit アクション ID 一覧

| action_id | 発生タイミング | 処理 |
|-----------|--------------|------|
| `task_confirm_yes` | 「タスクに追加」ボタン押下 | 期日抽出 → あり: 登録、なし: 日付ピッカー表示 |
| `task_confirm_no` | 「スキップ」ボタン押下 | メッセージをスキップ済みに更新 |
| `task_date_confirm` | 「この日付で登録」ボタン押下 | `state.values` から選択日付を取得して登録 |
| `task_no_date` | 「日付なしで登録」ボタン押下 | 期日なしで登録 |

日付ピッカーは `block_id: date_block`、`action_id: task_date_picker` で配置。
`task_date_confirm` 押下時に `payload.state.values.date_block.task_date_picker.selected_date` から日付を取得する。

## Notion 更新仕様

### updateKnowledgePage(pageId, patch)

```typescript
patch: {
  category?: string   // 'タスク' | '完了'
  dueDate?: string    // YYYY-MM-DD | undefined（省略時は更新しない）
}
```

Notion API: `pages.update` で `properties` のみ更新（`children` は変更しない）

### queryTasksDueSoon()

```typescript
// フィルター条件（今日〜2日後）
filter: {
  and: [
    { property: 'Category', select: { equals: 'タスク' } },
    { property: 'DueDate', date: { on_or_after: todayISO } },
    { property: 'DueDate', date: { on_or_before: twoDaysLaterISO } },
  ]
}
// 返却値
Array<{ id: string; title: string; dueDate: string; notionUrl: string }>
```

JST の日付計算: `new Date(Date.now() + 9 * 3600 * 1000).toISOString().slice(0, 10)`

## 期日抽出（Gemini）

```
モデル: gemini-2.5-flash
プロンプト:
  今日は {YYYY-MM-DD}（JST）です。
  以下のSlackメッセージから期限・締め切り・実施予定日を抽出してください。
  「今日」「明日」「来週月曜」「3/31」「3月31日」などをYYYY-MM-DD形式に変換してください。
  期限が見つからない場合は null を返してください。
  必ずJSON形式のみで返してください: {"dueDate": "YYYY-MM-DD"} または {"dueDate": null}
  メッセージ: {text}

パース: JSONから dueDate を取得。失敗・null の場合は undefined として扱う
```

## Slack メッセージ仕様

### タスク候補確認メッセージ

```
📌 このメッセージをタスクに追加しますか？
> {rawText の先頭40文字}...

[タスクに追加]  [スキップ]
```

### 日付ピッカーメッセージ（期日未検出時）

```
📅 いつまでにやりますか？
> {rawText の先頭40文字}...

[日付ピッカー(デフォルト: 翌日)]

[この日付で登録]  [日付なしで登録]
```

### タスク登録完了通知

```
// 期日あり
✅ タスクに登録しました（期日: MM/DD）
<@userId> さんの担当として Notion に保存しました。
[Notion で開く]

// 期日なし
✅ タスクに登録しました（期日なし）
<@userId> さんの担当として Notion に保存しました。
[Notion で開く]
```

### リマインダー通知

```
⏰ <@userId> 期限2日前: {title}（MM/DD まで）<notionUrl|Notionで開く>
⚠️ <@userId> 期限は明日: {title}（明日まで）<notionUrl|Notionで開く>
🔴 <@userId> 本日期限: {title}（今日中！）<notionUrl|Notionで開く>
```

### 完了通知

```
✅ タスク完了: {title}
<@userId> さんが完了しました。お疲れ様でした！
[Notion で開く]
```

## API エンドポイント設計

### app/api/slack/events/route.ts（変更点）

`handleMessage` に以下を追加（`processSingleMessage` 成功後）:

```typescript
// タスク検知（非同期・失敗しても継続）
if (pageId && detectTaskKeywords(rawText)) {
  void notifyTaskCandidate(channelId, ts, rawText, userId as string, pageId)
}

// 完了検知（スレッド返信のみ）
if (isReply && detectCompletionKeywords(rawText)) {
  void handleTaskCompletion(channelId, threadTs!)
}
```

### app/api/slack/actions/route.ts（変更点）

```typescript
} else if (actionId === 'task_confirm_yes') {
  const pending = await getPendingTask(channelId, messageTs)
  if (!pending) return NextResponse.json({ ok: true })
  await deletePendingTask(channelId, messageTs)
  await updateTaskMessage(channelId, messageTs, '⏳ 期日を確認中...')

  const dueDate = await extractDueDate(pending.rawText)
  if (dueDate) {
    // 期日あり → 即登録
    await registerTask(channelId, messageTs, pending, dueDate)
  } else {
    // 期日なし → 日付ピッカー表示
    await showDatePicker(channelId, messageTs, pending)
  }

} else if (actionId === 'task_confirm_no') {
  await updateTaskMessage(channelId, messageTs, 'スキップしました。')

} else if (actionId === 'task_date_confirm') {
  const selectedDate = payload.state?.values?.date_block?.task_date_picker?.selected_date
  const pending = await getDatePendingTask(channelId, messageTs)
  if (!pending) return NextResponse.json({ ok: true })
  await deleteDatePendingTask(channelId, messageTs)
  await registerTask(channelId, messageTs, pending, selectedDate ?? undefined)

} else if (actionId === 'task_no_date') {
  const pending = await getDatePendingTask(channelId, messageTs)
  if (!pending) return NextResponse.json({ ok: true })
  await deleteDatePendingTask(channelId, messageTs)
  await registerTask(channelId, messageTs, pending, undefined)
}
```

### app/api/cron/remind/route.ts（新規）

```
export const dynamic = 'force-dynamic'
GET /api/cron/remind

- 各タスクに対して:
    daysUntil = DueDate - today (JST)

    skip if isReminderSent(pageId, type)

    await markReminderSent(pageId, type)
```
