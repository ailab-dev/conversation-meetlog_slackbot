# 要件定義書 — フェーズ4 チャンネル参加時データ収集

最終更新: 2026-03-10

---

## 1. 背景・目的

フェーズ3までは過去データの取り込みにローカルスクリプト（`batch-import.ts`）が必要で、
開発者以外には実行できなかった。

フェーズ4では、ボットをチャンネルに追加するだけで Slack 上のボタン操作のみで
過去のメッセージを一括収集できる仕組みを提供する。

---

## 2. 保存対象の方針

| タイミング | 保存対象 |
|------------|----------|
| **過去メッセージ（一括収集時）** | チャンネルの **全メッセージ**（リアクションの有無を問わない） |
| **新着メッセージ（リアルタイム）** | 📌 / 💡 / ✅ の**リアクションが付いたもの**のみ（フェーズ1と同じ） |

過去メッセージを全件保存する理由：
- リアクションが付く前のやり取りにも有用な情報が含まれることが多い
- チャンネル全体のコンテキストをナレッジとして活用できる

---

## 3. トリガー

**ボット参加時（唯一のトリガー）**

- ボットを `/invite @knowledgeBot` でチャンネルに追加した瞬間に確認メッセージを自動送信する
- Slack イベント: `member_joined_channel`（参加したユーザーがボット自身である場合のみ）

---

## 4. 機能要件

### 4-1. 確認メッセージの送信

ボット参加時に、Block Kit のボタン付きメッセージをチャンネルに投稿する。

```
📥 このチャンネルの過去のメッセージをすべて保存しますか？

[はい]  [いいえ]
```

### 4-2. ボタン操作後の動作

| ボタン | 動作 |
|--------|------|
| [はい] | ボタンを削除し「収集を開始します...」と表示。バックグラウンドで収集処理を起動 |
| [いいえ] | ボタンを削除し「キャンセルしました」と表示 |

### 4-3. データ収集処理

- 対象: チャンネルの**全メッセージ履歴**（リアクション有無を問わない）
- 処理内容: `batch-import.ts` のエンリッチメント部分と同等
  - スレッド全文取得
  - フェーズ3エンリッチメント（URL・PDF・画像抽出）
  - Notion 保存（1,900文字チャンク分割）
  - Gemini Embedding → Upstash Vector 保存
  - Redis による重複チェック（既存データは再登録しない）
- 最低文字数フィルター: 5文字未満のメッセージはスキップ（スタンプのみ・空メッセージ等）

### 4-4. 完了通知

| タイミング | 通知内容 |
|------------|----------|
| 収集開始時 | 「収集を開始しました。完了後に通知します。」 |
| 収集完了時 | 「✅ 収集完了: X 件のメッセージを保存しました。」 |
| エラー発生時 | 「収集中にエラーが発生しました。」 |

---

## 5. 非機能要件

| 項目 | 要件 |
|------|------|
| Vercel タイムアウト対応 | チャンネル全履歴は数百〜数千件に及ぶため、1リクエストで処理しない。キュー（Upstash QStash）でチャンク処理する |
| 冪等性 | Redis 重複チェックにより既存データの再登録を防止 |
| ボット参加検知の精度 | ボット自身の参加のみ対象（一般メンバーの参加イベントは無視） |
| ボタン応答速度 | Slack の 3 秒タイムアウト内に HTTP 200 を返し、収集はバックグラウンドで実行 |
| 多重起動防止 | 同一チャンネルで収集が進行中の場合は「現在収集中です」と通知し二重起動しない |

---

## 6. Slack App 追加設定

| 設定項目 | 内容 |
|----------|------|
| Interactivity | 有効化。Request URL: `https://{domain}/api/slack/actions` |
| Event Subscriptions | `member_joined_channel` を追加 |
| Bot Token Scopes | 追加不要（既存スコープで対応可能） |

---

## 7. 将来拡張（対象外）

| 機能 | 概要 |
|------|------|
| スラッシュコマンド `/knowledge-collect` | ボット参加後に手動で再収集したい場合のコマンド |
| 収集範囲の指定 | 「過去 30日分だけ」などの期間指定オプション |
| 収集状況の確認 | 現在の収集件数や最終収集日時の確認 |
